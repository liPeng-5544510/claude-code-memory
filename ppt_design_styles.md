## 二、通用设计原则

### 核心原则（所有风格通用）

1. **结论先行** - 每页标题即核心观点
2. **数据情境化** - 有对比、有趋势、有基准
3. **信息可视化** - 图表优先于文字
4. **专业留白** - 内容占比 < 65%
5. **网格对齐** - 40px 基准网格

### 字体系统

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 封面标题 | 48-64px | Bold | 封面主标题 |
| H2 页面标题 | 28-36px | Bold | 页面大标题 |
| H3 模块标题 | 22-24px | Bold | 卡片/章节标题 |
| H4 小标题 | 18-20px | Bold | 列表/卡片标题 |
| P 正文 | 16-18px | Regular | 正文内容 |
| Data 数据 | 36-56px | Bold | 强调数据 |
| Sub 辅助 | 12-14px | Regular | 页码、来源 |

### 间距规范

```
页面边距: 40-60px
模块间距: 32-48px
卡片间距: 16-24px
卡片内边距: 20-32px
卡片圆角: 4-16px (视风格而定)
文字行距: 1.5-1.6
```

## 四、常用布局模式

| 模式 | 适用场景 | 说明 |
|------|---------|------|
| **居中卡片式** | 封面、结束页 | 中央内容卡片 + 渐变背景 |
| **左右分栏 5:5** | 对比展示 | 对称对比 |
| **左右分栏 4:6** | 图文混排 | 左文右图 |
| **左右分栏 3:7** | 强调视觉 | 大图配说明 |
| **三栏布局** | 三个要点 | 对称分布 |
| **四宫格** | 四个模块 | 2×2 网格 |
| **流程图** | 时间线/步骤 | 水平或垂直流程 |
| **矩阵** | 多维分析 | 优先级矩阵、SWOT |
| **Dashboard** | 数据看板 | 多指标展示 |
| **卡片列表** | 目录/要点 | 宽卡片垂直排列 |
| **中心辐射** | 核心概念展开 | 中心→四周 |
| **不对称对比** | 动感排版 | 非对称结构 |

## 五、SVG 技术约束

### 必须遵守

1. `viewBox="0 0 1280 720"`
2. 背景使用 `<rect>`
3. 文本换行用 `<tspan>`
4. 禁止 `<foreignObject>`
5. 禁止 `<style>` 和 `class`
6. 禁止 `rgba()`，用 HEX + opacity
7. 箭头用 `<polygon>` 三角形
8. 渐变用 `<defs><linearGradient>`

### PPT 兼容性黑名单

| 禁用元素 | 替代方案 |
|---------|---------|
| `<g opacity="...">` | 每个子元素单独设置透明度 |
| `clipPath` | 不使用裁剪 |
| `mask` | 不使用遮罩 |
| `textPath` | 使用普通 `<text>` |
| `animate*` | HTML可用，SVG禁用 |
| `marker-end` | 使用 `<polygon>` 箭头 |

## 六、HTML 全屏幻灯片特性

### 相比 PPT 的优势

- **丰富动画** - CSS动画、过渡效果
- **易于分享** - 链接 vs 文件
- **浏览器查看** - 无需软件
- **交互元素** - 点击、悬停等
- **响应式** - 适配不同屏幕

### 操作方式

| 按键 | 功能 |
|------|------|
| `→` / `空格` / `PageDown` | 下一页 |
| `←` / `PageUp` | 上一页 |
| `F` | 切换全屏 |
| `ESC` | 退出全屏 |
| `Home` | 第一页 |
| `End` | 最后一页 |

## 九、使用流程

1. **确定风格** - 根据内容和场景选择风格
2. **提取信息** - 从源文档提取关键内容
3. **选择布局** - 选择合适的页面布局模式
4. **填充内容** - 按模板填充内容
5. **检查质量** - 对照核对清单
6. **输出文件** - HTML (推荐) 或 SVG
